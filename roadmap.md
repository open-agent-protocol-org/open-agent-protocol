# OAP Roadmap (12 months)

## Months 1–3: Foundation
- Publish OAP v0.1 (draft)
- Collect feedback via Issues/Discussions
- Add 5–10 example agents
- Clarify runtime compliance requirements
- Draft OAP v0.2 proposal

## Months 4–6: Validation
- Release OAP v0.2
- Add `CONTRIBUTING.md` and contributor onboarding
- Add a minimal “reference runtime” plan (no full implementation required yet)
- Start permission taxonomy expansion + risk levels
- Run a small online builder sprint (community-driven)

## Months 7–9: Ecosystem
- Publish OAP v0.3 with event/trigger model + tool interface improvements
- Create a CLI skeleton (e.g., `oap init`, `oap validate`)
- Add a compatibility test checklist for runtimes
- Reach 50–100 community agents (mix of examples + real)

## Months 10–12: Institutionalization
- Propose OAP v1.0
- Draft governance model (working group + maintainers)
- Publish a reference implementation (minimal runtime) OR compliance test suite
- Prepare ecosystem documentation (best practices, security, permissions UX)
